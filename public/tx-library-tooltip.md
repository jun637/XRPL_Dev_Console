# 📚 Transaction Library 가이드

---

> 🧠 XRPL에는 이더리움처럼 스마트 컨트랙트가 없는 대신, **트랜잭션을 원장에 제출해 네트워크와 상호작용**합니다.
> **트랜잭션이 원장에 기록되는 순간**에 모든 상호작용이 일어나며, 결국 네트워크로 전송되는 구조는 공통된 `tx_json` 형식입니다.

---

## 1. 🔧 Transaction Library가 제공하는 것

1. **대표 트랜잭션 템플릿 모음**  
   - Payment, TrustSet, AMM, NFT, Credentials 등 XRPL에서 자주 쓰는 트랜잭션을 큐레이션했습니다.  
   - (100% 완전한 목록은 아니지만 실무 빈도가 높은 항목을 중심으로 계속 확장 중)

2. **문서 링크 + 언어별 참고 예제**  
   - 각 템플릿마다 공식 Docs, `xrpl.js`, `xrpl-py` 레퍼런스를 버튼으로 즉시 열 수 있습니다.  
   - 어떤 필드가 왜 필요한지, 언어별로 어떻게 직렬화하는지 빠르게 비교 가능합니다.

3. **Insert 버튼으로 즉시 사용**  
   - 템플릿을 따로 복사할 필요 없이 `Insert` 버튼 한 번으로 메인 화면의 Transaction Submit textarea에 채워집니다.  
   - 값만 조정해서 곧바로 서명/제출할 수 있어 실험 속도가 크게 빨라집니다.

---

## 2. 🚀 이렇게 활용해 보세요

> ✅ **아이디어 → 실행 속도 단축**  
> - 새 기능을 테스트할 때 `tx_json` 구조를 외울 필요 없이 템플릿을 불러와 핵심 필드만 수정하면 됩니다.

> 📑 **문서/SDK와 나란히 비교**  
> - 템플릿과 공식 레퍼런스를 동시에 열어두면 필드 설명과 실제 JSON을 한눈에 볼 수 있습니다.

> 🧩 **팀 온보딩 자료로 재활용**  
> - 초보자도 “어떤 트랜잭션이 있는지, 어떤 플래그를 켜야 하는지” 빠르게 이해할 수 있어 교육 자료로 적합합니다.
